===============================================================================
               RFLINK FRAME SIMULATIONS FOR TESTING IN HOME ASSISTANT
===============================================================================

This file contains various methods and payloads to simulate receiving 433MHz
radio frames via the rflink_ui integration, without any physical hardware.

-------------------------------------------------------------------------------
METHOD A: Using Home Assistant's native "python_script" integration
-------------------------------------------------------------------------------
1. Enable Python script support in your `configuration.yaml`:
   python_script:

2. Create a folder named `python_scripts` in your Home Assistant configuration
   directory (where `configuration.yaml` is located).

3. Create a file, e.g. `simulate_rf.py` in that folder, with one of the simulation
   code snippets below.

4. Reload Python Scripts in Home Assistant.

5. Go to Developer Tools -> Services, find the service `python_script.simulate_rf`
   and click "Call Service".

-------------------------------------------------------------------------------
METHOD B: Execution via Developer Tools (Services)
-------------------------------------------------------------------------------
Some custom python console integrations (e.g. pyscript) allow running Python
code directly. Otherwise, Method A is the official native method.
-------------------------------------------------------------------------------


-------------------------------------------------------------------------------
1. Simulating a Switch / Remote Control Button
-------------------------------------------------------------------------------
To simulate receiving a press of a button (ON or OFF) on a remote control or
switch (Test Device ID: Unitec_1a4a_4):

# To Turn ON
device_id = "Unitec_1a4a_4"
hass.helpers.dispatcher.dispatcher_send(
    f"rflink_update_{device_id}", 
    {"CMD": "ON"}
)

# To Turn OFF
device_id = "Unitec_1a4a_4"
hass.helpers.dispatcher.dispatcher_send(
    f"rflink_update_{device_id}", 
    {"CMD": "OFF"}
)

-------------------------------------------------------------------------------
2. Simulating a Temperature and Humidity Sensor
-------------------------------------------------------------------------------
To simulate receiving weather data from a sensor (ID: Oregon_0A4C):

# Simulated data:
# TEMP = "00ba" (00ba hex = 186 dec -> converted by integration to 18.6°C)
# HUM = "40" (40% Humidity)
# BAT = "OK" (Battery level OK)

device_id = "Oregon_0A4C"
hass.helpers.dispatcher.dispatcher_send(
    f"rflink_update_{device_id}", 
    {
        "ID": "0A4C",
        "TEMP": "00ba", 
        "HUM": "40", 
        "BAT": "OK"
    }
)

-------------------------------------------------------------------------------
3. Simulating Gateway USB Connection State Changes
-------------------------------------------------------------------------------
To simulate connection or disconnection of the RFLink USB key itself.
Replace <ENTRY_ID> with your actual config_entry ID:

# Simulate gateway CONNECTED (Status: Connected)
entry_id = "<ENTRY_ID>"
hass.helpers.dispatcher.dispatcher_send(
    f"rflink_connection_{entry_id}", 
    True
)

# Simulate gateway DISCONNECTED (Status: Disconnected)
entry_id = "<ENTRY_ID>"
hass.helpers.dispatcher.dispatcher_send(
    f"rflink_connection_{entry_id}", 
    False
)
